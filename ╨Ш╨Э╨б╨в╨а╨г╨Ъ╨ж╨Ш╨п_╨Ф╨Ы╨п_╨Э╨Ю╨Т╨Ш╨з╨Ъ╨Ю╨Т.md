# Инструкция по запуску для новичков

Эта инструкция рассчитана на человека, который никогда раньше не запускал подобные проекты. Идём строго по шагам, ничего не пропускаем. Всего шагов: 7. Время: 30–40 минут в первый раз.

Если на каком-то шаге появится ошибка — не паникуйте, в конце есть раздел «Частые проблемы».

---

## Шаг 1. Установите Node.js

Node.js — это программа, без которой проект не запустится на компьютере.

1. Откройте [nodejs.org](https://nodejs.org)
2. Скачайте версию **LTS** (зелёная кнопка, она самая стабильная)
3. Запустите установщик и нажимайте «Далее» до конца, ничего не меняя
4. После установки откройте терминал:
   - **Windows**: нажмите Win+R, введите `cmd`, нажмите Enter
   - **Mac**: откройте приложение «Терминал» (Spotlight → набрать «Terminal»)
5. Введите команду и нажмите Enter:
   ```
   node --version
   ```
   Если видите что-то вроде `v20.11.0` — всё установилось правильно. Если терминал пишет «команда не найдена» — перезапустите компьютер и попробуйте снова.

## Шаг 2. Распакуйте проект

1. Скачайте файл `zaselenie-app.zip`, который я прислал
2. Распакуйте его в удобное место, например на Рабочий стол (правой кнопкой мыши → «Извлечь всё» на Windows, или просто двойной клик на Mac)
3. У вас должна получиться папка `zaselenie-app` с файлами внутри

## Шаг 3. Создайте проект в Firebase

Firebase — это бесплатный сервис от Google, который будет хранить все ваши данные (помещения, статусы, пользователей) и обеспечивать вход в систему. Это и есть тот самый «бэкенд».

### 3.1 Регистрация проекта

1. Откройте [console.firebase.google.com](https://console.firebase.google.com)
2. Войдите через любой Google-аккаунт (если нет — создайте, это бесплатно)
3. Нажмите **«Создать проект»** (Create project / Add project)
4. Введите название, например `zaselenie-objekta` (можно русскими буквами, не принципиально)
5. На вопрос про Google Analytics — можно отключить переключатель, он не нужен для этого проекта
6. Нажмите «Создать проект» и подождите 30-60 секунд, пока Google его подготовит

### 3.2 Включите вход по email и паролю

1. В левом меню найдите **Build → Authentication**
2. Нажмите **«Get started»**
3. В списке способов входа найдите **«Email/Password»**, нажмите на неё
4. Включите первый переключатель (Enable) и нажмите **«Save»**

### 3.3 Создайте базу данных

1. В левом меню найдите **Build → Firestore Database**
2. Нажмите **«Create database»**
3. Выберите регион — берите ближайший к себе (например, `eur3 (europe-west)` для Европы)
4. Выберите режим **«Start in production mode»** (это безопаснее — конкретные правила доступа мы зададим отдельно на шаге 5)
5. Нажмите **«Create»**

### 3.4 Получите ключи доступа для приложения

1. В левом меню нажмите на шестерёнку рядом с «Project Overview» → **«Project settings»**
2. Прокрутите вниз до раздела **«Your apps»**
3. Нажмите на значок `</>`  (Web app)
4. Введите любое имя приложения, например `shahmatka-web`
5. Галочку «Firebase Hosting» можно не ставить
6. Нажмите **«Register app»**
7. Появится кусок кода вида:
   ```js
   const firebaseConfig = {
     apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXX",
     authDomain: "zaselenie-objekta.firebaseapp.com",
     projectId: "zaselenie-objekta",
     storageBucket: "zaselenie-objekta.appspot.com",
     messagingSenderId: "123456789012",
     appId: "1:123456789012:web:abcdef1234567890"
   };
   ```
   **Не закрывайте эту страницу** — этот код понадобится на следующем шаге. Можно скопировать его в любой текстовый файл, чтобы не потерять.

## Шаг 4. Впишите ключи в проект

1. Откройте папку `zaselenie-app`, которую распаковали на шаге 2
2. Зайдите в папку `src`
3. Откройте файл `firebase.js` любым текстовым редактором (подойдёт обычный Блокнот / TextEdit, но удобнее — бесплатная программа [VS Code](https://code.visualstudio.com))
4. Вы увидите такой блок:
   ```js
   const firebaseConfig = {
     apiKey: 'ВАШ_API_KEY',
     authDomain: 'ВАШ_PROJECT_ID.firebaseapp.com',
     projectId: 'ВАШ_PROJECT_ID',
     storageBucket: 'ВАШ_PROJECT_ID.appspot.com',
     messagingSenderId: 'ВАШ_SENDER_ID',
     appId: 'ВАШ_APP_ID'
   };
   ```
5. Замените все значения на свои, скопированные с шага 3.4. Должно получиться примерно так:
   ```js
   const firebaseConfig = {
     apiKey: 'AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXX',
     authDomain: 'zaselenie-objekta.firebaseapp.com',
     projectId: 'zaselenie-objekta',
     storageBucket: 'zaselenie-objekta.appspot.com',
     messagingSenderId: '123456789012',
     appId: '1:123456789012:web:abcdef1234567890'
   };
   ```
6. Сохраните файл (Ctrl+S или Cmd+S)

**Важно:** кавычки `'...'` оставьте на месте, меняйте только то, что внутри них.

## Шаг 5. Опубликуйте правила безопасности

Это критически важный шаг. Без него обычные пользователи смогут менять структуру объекта в обход интерфейса, а данные не будут защищены.

1. Откройте терминал (см. шаг 1, пункт 4)
2. Установите инструмент командной строки Firebase. Введите команду:
   ```
   npm install -g firebase-tools
   ```
   Подождите, пока установится (1-2 минуты)
3. Авторизуйтесь:
   ```
   firebase login
   ```
   Откроется браузер — войдите тем же Google-аккаунтом, что использовали в Firebase Console, и разрешите доступ
4. В терминале перейдите в папку проекта. Если вы распаковали её на Рабочий стол, команда будет такой:

   **Windows:**
   ```
   cd Desktop\zaselenie-app
   ```
   **Mac:**
   ```
   cd ~/Desktop/zaselenie-app
   ```
5. Свяжите папку с вашим Firebase-проектом:
   ```
   firebase use --add
   ```
   Выберите из списка ваш проект (например `zaselenie-objekta`) стрелками и нажмите Enter. На вопрос про алиас — просто нажмите Enter (оставит `default`)
6. Опубликуйте правила:
   ```
   firebase deploy --only firestore:rules
   ```
   Должно появиться сообщение `✔ Deploy complete!`

## Шаг 6. Запустите приложение на компьютере

1. В том же терминале (вы уже должны быть внутри папки `zaselenie-app`) выполните:
   ```
   npm install
   ```
   Это скачает все необходимые библиотеки. Может занять 1-3 минуты, в терминале будет «бежать» текст — это нормально.

2. После того как установка завершилась, запустите проект:
   ```
   npm run dev
   ```
3. В терминале появится строка вида:
   ```
   ➜  Local:   http://localhost:5173/
   ```
4. Откройте эту ссылку в браузере (зажмите Ctrl и кликните по ней, либо скопируйте в адресную строку)
5. Вы увидите экран входа в систему

**Чтобы остановить приложение** — вернитесь в терминал и нажмите `Ctrl+C`.
**Чтобы запустить снова** — повторите команду `npm run dev` (команду `npm install` второй раз делать не нужно).

## Шаг 7. Первый вход и настройка

1. На экране входа нажмите **«Регистрация»**
2. Введите своё имя, любой email (например `admin@stroika.ru`, реальная почта необязательна на этом этапе, главное — запомнить) и пароль (минимум 6 символов)
3. Нажмите **«Подать заявку на доступ»**

Поскольку вы — первый зарегистрированный пользователь, система автоматически сделает вас **администратором** и сразу даст доступ, без ожидания.

4. Перейдите во вкладку **«Структура»** и создайте первый корпус, добавьте этажи и помещения
5. Дайте остальным сотрудникам ссылку (после публикации в интернет, см. ниже) — они смогут зарегистрироваться сами, а вы будете одобрять их во вкладке **«Пользователи»**

## Что дальше: публикация в интернет

Пока что приложение работает только на вашем компьютере (`localhost`). Чтобы сотрудники могли открывать его с телефонов из любой точки, нужно опубликовать его в интернет. Это бесплатно через тот же Firebase.

1. В терминале (в папке проекта) выполните:
   ```
   npm run build
   ```
   Дождитесь завершения (10-30 секунд)
2. Опубликуйте:
   ```
   firebase deploy --only hosting
   ```
3. В конце появится ссылка вида:
   ```
   Hosting URL: https://zaselenie-objekta.web.app
   ```

Эта ссылка постоянная — отправьте её сотрудникам. Они откроют её с телефона, а через меню браузера (обычно три точки → «Добавить на главный экран») смогут установить приложение как обычное мобильное.

**Если позже что-то поменяете в коде** — чтобы изменения попали в интернет, нужно заново выполнить `npm run build`, а затем `firebase deploy --only hosting`.

## Частые проблемы

**«npm: команда не найдена» / «npm is not recognized»**
Node.js не установился или нужен перезапуск компьютера. Повторите шаг 1.

**«firebase: команда не найдена» после установки firebase-tools**
Закройте терминал полностью и откройте заново — иногда команда не подхватывается сразу.

**При входе в приложение пишет ошибку или белый экран**
Скорее всего, неверно скопированы ключи на шаге 4. Откройте `src/firebase.js` и сверьте значения ещё раз — особенно `projectId`, в нём не должно быть лишних пробелов или незакрытых кавычек.

**«Missing or insufficient permissions» при попытке что-то сохранить**
Правила безопасности не опубликованы — вернитесь к шагу 5, команда `firebase deploy --only firestore:rules`.

**Регистрация второго пользователя не даёт доступ сразу — это норма?**
Да, это правильное поведение. Только первый пользователь становится админом автоматически. Второй и все следующие будут ждать, пока администратор одобрит их во вкладке «Пользователи».

**Хочу сменить порт 5173 на другой**
Откройте файл `vite.config.js`, найдите строку `port: 5173` и замените на нужный номер.
